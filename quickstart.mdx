This guide helps you send your **first request** to Riskor in minutes.  
It focuses on the essentials only: **API key**, **Bearer auth**, **Sandbox vs Production**, and minimal **cURL / Node.js / Python** examples.

<Callout>
Looking for scoring semantics, thresholds, or detailed response fields? See <a href="/scoring-and-thresholding">Scoring & Thresholding (0–3)</a> and <a href="/response-schema">Response Schema</a>.
</Callout>

---

## Prerequisites

- A Riskor account and access to the dashboard.
- Network access to the Riskor HTTPS endpoint.
- One of: cURL, Node.js (fetch), or Python (requests).

---

## Quick Path

<Steps>
  <Step title="Create an API Key">
    1. Open the Riskor dashboard and generate a key for **Sandbox** first.  
    2. Copy your key and keep it secure — you will use it as a **Bearer token**.

    <Callout>
    For security guidance (key rotation, least privilege, 401/403), see <a href="/authentication">Authentication</a>.
    </Callout>
  </Step>

  <Step title="Choose Environment: Sandbox vs Production">
    - **Sandbox**: Safe for testing. Same request shape, non-production quotas/latency.  
    - **Production**: Live traffic with higher quotas and SLAs (subject to your plan).
    <br />
    Switch environments by changing the **base URL** and using the corresponding **API key**.
  </Step>

  <Step title="Send Your First Request">
    Use one of the minimal examples below. Replace:
    - <code>RISKOR_API_KEY</code> — with your key
    - <code>BASE_URL</code> — with your environment URL (Sandbox or Production)
    - <code>text</code> — with the content you want to analyze

    <Tabs>
      <Tab title="cURL">
        ```bash
        export RISKOR_API_KEY="sk_sandbox_xxx"
        export BASE_URL="https://sandbox.api.riskor.ai"

        curl -X POST "$BASE_URL/text-moderate" \
          -H "Authorization: Bearer $RISKOR_API_KEY" \
          -H "Content-Type: application/json" \
          -d '{
            "text": "I can double your money in 30 days, guaranteed.",
            "language": "en"
          }'
        ```
      </Tab>

      <Tab title="Node.js">
        ```javascript
        const BASE_URL = "https://sandbox.api.riskor.ai";
        const API_KEY = process.env.RISKOR_API_KEY; // set in your env

        async function run() {
          const res = await fetch(`${BASE_URL}/text-moderate`, {
            method: "POST",
            headers: {
              "Authorization": `Bearer ${API_KEY}`,
              "Content-Type": "application/json"
            },
            body: JSON.stringify({
              text: "I can double your money in 30 days, guaranteed.",
              language: "en"
            })
          });

          if (!res.ok) {
            console.error("HTTP", res.status, await res.text());
            return;
          }

          const data = await res.json();
          console.log(data);
          // data.final_score is an integer 0–3
          // Map it to your product action on your side.
        }

        run().catch(console.error);
        ```
      </Tab>

      <Tab title="Python">
        ```python
        import os, requests, json

        BASE_URL = "https://sandbox.api.riskor.ai"
        API_KEY = os.getenv("RISKOR_API_KEY")

        payload = {
            "text": "I can double your money in 30 days, guaranteed.",
            "language": "en"
        }

        r = requests.post(
            f"{BASE_URL}/text-moderate",
            headers={
                "Authorization": f"Bearer {API_KEY}",
                "Content-Type": "application/json"
            },
            data=json.dumps(payload)
        )

        if r.status_code != 200:
            print("HTTP", r.status_code, r.text)
        else:
            data = r.json()
            print(data)
            # data["final_score"] in {0,1,2,3}
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Minimal Request Shape

Keep your first call simple:

| Field     | Type   | Required | Notes                                   |
|-----------|--------|----------|-----------------------------------------|
| `text`    | string | yes      | The text to analyze.                    |
| `language`| string | no       | Language hint (e.g., `"en"`, `"zh-TW"`). If omitted, server may infer. |

For full request/response structure, see **<a href="/text-moderate">Text Moderate API</a>** and **<a href="/response-schema">Response Schema</a>**.

## Bearer Authentication (Header)

Send your API key via the `Authorization` header:

```http
Authorization: Bearer YOUR_API_KEY
If you receive 401 or 403, review <a href="/authentication">Authentication</a>.

What To Do With the Score
Your response includes a discrete final_score in 0–3.
Map it to product actions on your side (e.g., Pass / Warn / Review / Block).
For decision mapping patterns, see <a href="/scoring-and-thresholding">Scoring & Thresholding (0–3)</a>.
